# TD credit-card parser

## Support boundary

`TdCreditCardParser` version `1.0.0` supports the text-extractable TD Canada
Trust credit-card layout represented by the reviewed product-plan findings and
the entirely synthetic `synthetic-td-v1` fixture. It requires:

- a TD issuer marker;
- an explicit statement-period range;
- the `TRANSACTION DATE / POSTING DATE / ACTIVITY DESCRIPTION / AMOUNT` table
  header; and
- transaction rows with abbreviated month/day transaction and posting dates.

Issuer or table markers that do not match this grammar fail closed with an
actionable unsupported-layout error. Blank/image-only extraction fails as a
scanned-document error; OCR is not attempted.

## Section and row behavior

The adapter preserves transaction and posting dates, inferring their year only
when exactly one date falls inside the statement period. It processes rows
across repeated page headers, ignores page numbers, side summaries, payment
slips, rewards content, and legal text, and supports explicit multi-line
description and foreign-currency continuations.

Amounts are parsed directly into signed integer CAD cents. Commas, dollar
symbols, parentheses, leading minus signs, and `CR` suffixes are handled without
floating-point arithmetic; comma grouping must be conventional thousands
grouping or the row is rejected. `PAYMENT - THANK YOU` becomes `payment`; negative
returns/credits become `refund`; interest, fees, and cash advances are classified
after all description continuations are assembled; remaining debits are purchases.
Foreign rows retain the charged CAD
amount and optionally attach an uppercase currency, signed original minor-unit
amount, and a positive fixed-eight-place `Decimal` exchange rate. Duplicate or
conflicting foreign continuations fail closed instead of overwriting prior data.

Identical canonical rows receive occurrence indexes `0`, `1`, and so on in
source order. The parser deliberately does not create fingerprints or own key
material. BE-15 must supply the local secret (at least 32 bytes) when it calls
the BE-13 HMAC fingerprint primitive.

## Reconciliation

TD summary values for payments/credits, purchases/other charges, interest, and
fees are normalized into signed credit and positive debit totals. Reconciliation
checks the net activity, aggregate debit/credit totals, and each available
purchases, interest, fees, and credits section independently with a one-cent
tolerance. This prevents equal-and-opposite section errors from cancelling. A
missing summary or any section delta beyond one cent returns
`needs_review`; detection confidence never overrides arithmetic reconciliation.

## Fixtures and privacy

[`fixtures/statements/td/manifest.json`](../../fixtures/statements/td/manifest.json)
records layout versions, page counts, period shape, and covered cases. The PDFs
are deterministically generated by `generate_fixtures.py`; all names and
transactions are explicitly synthetic. They contain no person, address,
barcode, unmasked account number, or real transaction history. Expected JSON
contains only canonical structured output and integer cents.

Private samples may be exercised only from an ignored external path. They must
never be copied into this repository, test output, logs, snapshots, or Graphify
input.
